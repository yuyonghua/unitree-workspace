## 技术选型理由

### 为什么用 wty-yy 的 RL 方案而不是官方固件？

| 对比项 | 官方 Sport Mode | wty-yy RL 方案 |
|-------|----------------|----------------|
| 控制方式 | 位置控制，行走姿态固定 | 神经网络，直接力矩控制 |
| 地形适应 | 一般（需要遥控精细操作） | 强（盲上楼梯、乱地） |
| 最高速度 | ~1.5m/s | ~4m/s |
| 扩展性 | 封闭，无法改算法 | 完全开源，可定制 |
| Sim2Real | N/A | 已验证，误差小 |

### 为什么用 Web 可视化而不是 RViz？

| 对比项 | Web 网页 | RViz |
|--------|---------|------|
| 多端访问 | ✅ 任何设备浏览器 | ❌ 只能 Linux |
| 安装依赖 | ❌ 无需安装 | ❌ 需要 ROS2 环境 |
| 用户友好 | ✅ 美观直观 | ❌ 开发者工具 |
| 实时性能 | ✅ 60fps Canvas/Three.js | ✅ 60fps OpenGL |
| 开发成本 | 较高（需要自研） | 低（开箱即用） |
