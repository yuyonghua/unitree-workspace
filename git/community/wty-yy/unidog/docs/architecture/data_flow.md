## 二、导航模式下的数据流

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    导航模式数据流 (Navigation Mode)                       │
└─────────────────────────────────────────────────────────────────────────┘

用户操作 (点击地图目标点)
      │
      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  UniDog Web (浏览器)                                                       │
│  - 点击地图某处 ──► 获取坐标 (x, y)                                        │
│  - 发送目标点给 MCP Server                                                 │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │ HTTP POST
┌──────────────────────────────▼──────────────────────────────────────────┐
│  MCP Server (Python)                                                       │
│  - 解析自然语言命令 (如果语音)                                              │
│  - 调用 navigate_to(x, y) 工具                                            │
│  - 通过 ROS2 Action 发送给 Nav2                                            │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │ /navigate_to_pose (Action)
┌──────────────────────────────▼──────────────────────────────────────────┐
│  Nav2 (ROS2)                                                              │
│                                                                             │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                         Global Planner                            │   │
│  │  输入: 目标点 (x,y)    输出: 全局路径 (Path)                      │   │
│  │  算法: NavFn / SMAC                                              │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                               │                                           │
│                               ▼                                           │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                         Local Planner (DWB)                       │   │
│  │  输入: 全局路径 + 雷达点云   输出: /cmd_vel (Twist)               │   │
│  │  功能: 轨迹跟踪 + 实时避障 (DWA 算法)                             │   │
│  │  避障: 检测前方 70° 范围内的障碍物, 动态重规划                     │   │
│  └──────────────────────────────────────────────────────────────────┘   │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │ /cmd_vel
          ┌────────────────────┼────────────────────┐
          │                    │                    │
          ▼                    ▼                    ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  wty-yy C++     │  │  Point-LIO     │  │  EKF Fusion    │
│  go2_ctrl       │  │                 │  │                 │
│                 │  │  持续发布       │  │  融合:          │
│  接收 /cmd_vel   │  │  Odometry      │  │  IMU            │
│  RL 推理        │  │                 │  │  里程计          │
│  发布 /lowcmd   │  │                 │  │  Point-LIO      │
│                 │  │                 │  │  → 精确位姿      │
└────────┬────────┘  └─────────────────┘  └─────────────────┘
         │
         ▼
┌─────────────────┐
│  关节电机执行    │
│  狗自主行走     │
└─────────────────┘
```

## 三、语音控制模式下的数据流

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    语音控制数据流 (Voice Control Mode)                    │
└─────────────────────────────────────────────────────────────────────────┘

用户语音
"小GO, 去A点巡检一下"
      │
      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  Voice Client (Python)                                                    │
│                                                                             │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │  Whisper STT                                                       │   │
│  │  麦克风 ──► 音频 ──► whisper.cpp ──► 文本 "去A点巡检一下"         │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                               │                                            │
│                               ▼                                            │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │  LLM API (Claude / GPT-4o)                                        │   │
│  │  System Prompt: 你是一个机器狗控制助手...                          │   │
│  │  Tools: navigate_to, set_velocity, emergency_stop...              │   │
│  │  输入: "去A点巡检一下"                                            │   │
│  │  输出: Tool Call { "navigate_to": { "x": 5.2, "y": -1.3 } }     │   │
│  └──────────────────────────────────────────────────────────────────┘   │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │ Tool Call
┌──────────────────────────────▼──────────────────────────────────────────┐
│  MCP Server (Python)                                                       │
│  - 接收 LLM 的工具调用请求                                                │
│  - 调用对应底层接口                                                       │
│  - 返回执行结果给 LLM                                                     │
│                                                                             │
│  如果是 navigate_to(x, y):                                                │
│    1. 查询位置数据库, 获取 A点坐标                                         │
│    2. 调用 Nav2 Action 发送目标点                                         │
│    3. 等待 Nav2 返回 "已到达"                                             │
│    4. 返回结果给 LLM                                                      │
│                                                                             │
│  如果是 emergency_stop():                                                  │
│    1. 绕过所有层级, 直接发 /cmd_vel = (0, 0, 0)                          │
│    2. 发 /lowcmd 置零力矩                                                  │
│    3. 返回 "已停止" 给 LLM                                                │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │
                               ▼
           (后续同导航模式数据流)
```

## 四、DDS Topic 完整清单

| Topic 名称 | 类型 | 频率 | 发布者 | 订阅者 | 说明 |
|-----------|------|------|-------|-------|------|
| `/lowstate` | LowState | 500Hz | 仿真器/真狗 | wty-yy C++ | 关节状态+IMU |
| `/lowcmd` | LowCmd | 500Hz | wty-yy C++ | 仿真器/真狗 | 目标力矩 |
| `/utlidar/cloud` | PointCloud2 | 10Hz | 雷达驱动 | Point-LIO | 雷达原始点云 |
| `/odom` | Odometry | 100Hz | Point-LIO | Nav2, WebSocket | 里程计 |
| `/map` | OccupancyGrid | 1Hz | Point-LIO | Nav2, WebSocket | 地图栅格 |
| `/cmd_vel` | Twist | 50Hz | Nav2, WebSocket | wty-yy C++ | 速度指令 |
| `/map_points` | PointCloud2 | 10Hz | Point-LIO | WebSocket | 建图点云 |
| `/joint_states` | JointState | 50Hz | wty-yy C++ | WebSocket | 关节角度 |

## 五、WebSocket 消息格式

```python
# 浏览器 ←── WebSocket ──→ 服务端

# 1. 雷达点云 (二进制, 高频)
{
    "type": "pointcloud",
    "data": Float32Array,  # [x, y, z, x, y, z, ...] 单位: 米
    "count": 24000,        # 点数量
    "timestamp": 1711000000
}

# 2. 机器狗位置 (JSON, 中频)
{
    "type": "odom",
    "position": {"x": 1.23, "y": -0.45, "z": 0.0},
    "orientation": {"roll": 0.01, "pitch": 0.02, "yaw": 0.15},
    "timestamp": 1711000000
}

# 3. 地图栅格 (二进制, 低频)
{
    "type": "gridmap",
    "width": 512,
    "height": 512,
    "resolution": 0.05,  # 米/像素
    "origin": {"x": -12.8, "y": -12.8},
    "data": Uint8Array  # 0=未知, 1=空闲, 2=占用
}

# 4. 遥控指令 (浏览器 → 服务端)
{
    "type": "cmd_vel",
    "vx": 0.5,      # m/s
    "vy": 0.0,      # m/s
    "omega": 0.2    # rad/s
}

# 5. 状态上报 (服务端 → 浏览器)
{
    "type": "state",
    "battery": 85,           # %
    "temperature": 42,        # °C
    "signal": "strong",      # 信号强度
    "status": "mapping",     # 状态: mapping/navigating/standing
    "odom_distance": 127.5    # 里程计, 米
}
```
